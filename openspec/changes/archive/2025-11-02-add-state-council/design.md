## Context
基於現有的常任理事會系統架構，建立一個平行運作的國務院系統。國務院將擁有獨立的治理結構、政府帳戶，並提供多部門管理功能。系統需要與現有經濟體系整合，但保持獨立的運作邏輯。

## Goals / Non-Goals
- Goals:
  - 建立完整的國務院治理系統
  - 提供分部門的面板管理介面
  - 實現定期福利發放和稅收徵收
  - 提供身分管理和貨幣政策工具
- Non-Goals:
  - 不與現有理事會系統合併
  - 不修改現有經濟體系基本規則

## Decisions
- Decision: 採用與理事會類似的面板架構，但增加部門分頁功能
  - Reason: 使用者已熟悉理事會操作模式，降低學習成本
  - Alternatives considered: 獨立視窗介面、純指令操作
- Decision: 各部門設立獨立政府帳戶
  - Reason: 便於資金管理和審計追蹤
  - Alternatives considered: 統一帳戶分項管理
- Decision: 使用身分組進行權限控制
  - Reason: 利用 Discord 既有的角色系統，便於管理
  - Alternatives considered: 獨立權限系統

## Risks / Trade-offs
- 資料庫結構複雜度增加 → 需要仔細設計以避免效能問題
- 帳戶數量增加 → 需要完善的轉帳記錄和審計機制
- 權限管理複雜化 → 需要清晰的權限配置介面

## Migration Plan
1. 建立國務院資料庫結構
2. 實作基礎服務層
3. 開發面板介面
4. 逐步實現各部門功能
5. 測試與整合驗證

- 各部門間的資金流動是否需要特殊審批？
- 疑犯身分的移除流程是否需要上訴機制？
